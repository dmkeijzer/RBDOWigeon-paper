<h1> Repository for Reliability-Based Design Optimization of an eVTOL</h1>

## Introduction

This repository contains the models, optimisation setup, and post‑processing tools used in the Design Synthesis Exercise for a long‑range electric VTOL (eVTOL) concept. The codebase brings together aerodynamic and performance models, structural and weight estimation, and mission analysis to evaluate and optimise candidate designs. In addition to deterministic analyses, the repository supports Monte Carlo Simulation (MCS) and Reliability‑Based Design Optimisation (RBDO) to capture the impact of uncertainty in key mission and design variables. The included plotting utilities make it straightforward to visualise optimisation results, compare stochastic and deterministic performance, and inspect the underlying statistical assumptions.



## Compatibility
-  The easiest method to run this repository is to use the docker image or build file provided. 
- All Python Package Index dependencies listed in requirements.txt (install with `pip install -r requirements.txt`)
- Tests written and stored in test directory with pytest

## Plotting and Post‑Processing

The `plot_scripts` directory contains convenience scripts for visualising optimisation and MCS/RBDO results and for inspecting the stochastic mission parameters used in the analyses.

### Plotting prerequisites

- **Python environment**: Ensure you have created a Python 3.12 environment and installed all dependencies listed in `requirements.txt` (for example with `pip install -r requirements.txt`).
- **Working directory**: All example commands below assume you run them from the repository root:

  ```bash
  cd /path/to/HonoursRepo
  python plot_scripts/<script_name>.py
  ```

  The scripts themselves change the working directory to the repository root, so running them from elsewhere is not recommended.
- **Result locations**: For Monte Carlo and baseline/best‑design results, the scripts rely on paths configured in `input/environment_variables_plotting.yml`. In particular, the keys `mcs.dir` and `baseline.dir` must point to the folders containing the corresponding optimisation/MCS output.

### Scripts in `plot_scripts`

#### `plot_parameters.py`

- **Purpose**: Visualises RBDO‑based optimisation results and baseline performance using `modules.plotting.npz_reader` and CSV‑based readers. It produces plots of design parameters, energy distribution over mission phases, and energy pie charts for the baseline.
- **How to run**:

  ```bash
  python plot_scripts/plot_parameters.py
  ```

- **Inputs**:
  - A configured `input/environment_variables_plotting.yml` whose `mcs.dir` and `baseline.dir` entries point to valid result directories.
  - `.npz` files and baseline CSV outputs generated by the optimisation/MCS runs and stored in the directories referenced in the YAML file.
- **Outputs**:
  - Figures stored in the `plots` subdirectories of the configured MCS and baseline result folders.
  - Terminal messages confirming successful generation of RBDO and baseline plots.

#### `plot_comparison.py`

- **Purpose**: Compares stochastic (MCS) and deterministic results using helper functions from `modules.plotting.compare_MCS_deter` and `modules.plotting.MCS_metric_deterministic_reader`. Typical outputs include PDF/CDF plots, weight comparisons, and energy distribution pie charts.
- **How to run**:

  ```bash
  python plot_scripts/plot_comparison.py
  ```

- **Inputs**:
  - Metric data and `.npy` files produced by the MCS workflows and deterministic/baseline analyses.
  - The same result directories and configuration as used for `plot_mcs.py` and `create_MCS_metric.py`.
- **Outputs**:
  - Comparison plots saved alongside other plots in the relevant `plots` folders.

#### `plot_mcs.py`

- **Purpose**: Processes existing MCS and baseline results to:
  - Plot the convergence of the MCS energy metric.
  - Assemble sorted mission‑sample tables for both MCS and baseline.
  - Compute and save correlation matrices between energy and key mission/trajectory parameters.
- **How to run**:

  ```bash
  python plot_scripts/plot_mcs.py
  ```

- **Inputs**:
  - `input/environment_variables_plotting.yml` with:
    - `mcs.dir`: directory containing `mcs_convergence<label>.npy`, `sample_history<label>.npy`, and `mission_results<label>.npy`.
    - `baseline.dir`: directory containing the corresponding `mcs_metric` arrays (see `create_MCS_metric.py` below).
- **Outputs**:
  - Convergence plot PDF in the `plots` subfolder of the MCS directory.
  - `mission_samples_sorted<label>.xlsx` files in the `tables` subfolders of both MCS and baseline directories for easy inspection in spreadsheet software.
  - `correlation_matrix<label>.txt` files in the same `tables` folders, containing the correlation matrices between energy and the main mission parameters.

#### `create_MCS_metric.py`

- **Purpose**: Regenerates MCS metrics for the baseline mission using a stored, pickled mission object. This script is typically run once per baseline configuration and its outputs are used by other plotting scripts.
- **How to run**:

  ```bash
  python plot_scripts/create_MCS_metric.py
  ```

- **Inputs**:
  - `input/environment_variables_plotting.yml` where `baseline.dir` points to a directory containing a baseline mission pickle file named `Mission_class<label>.pkl`.
  - The mission pickle is produced by the main analysis pipeline and must exist before running this script.
- **Outputs**:
  - A `mcs_metric` subfolder inside the baseline directory.
  - Within `mcs_metric`, the script saves:
    - `mcs_metric_samples<label>.npy` and `mcs_metric_results<label>.npy`, containing samples and mission results.
    - Several `.pkl` files for random variables such as energy, time, power, thrust, and phase‑specific energies.
  - These files are prerequisites for some of the analysis performed in `plot_mcs.py` and `plot_comparison.py`.

#### `plot_stochastic_mission_parameters.py`

- **Purpose**: Provides standalone visualisations of the probability distributions assumed for key stochastic mission parameters (for example transition heights and mission range). It draws random samples and plots histograms overlaid with the corresponding theoretical probability density functions.
- **How to run**:

  ```bash
  python plot_scripts/plot_stochastic_mission_parameters.py
  ```

- **Inputs**:
  - Only standard numerical and plotting libraries (NumPy, SciPy, Matplotlib, Seaborn). It does not rely on the main optimisation results.
- **Outputs**:
  - PDF files saved to the user's `Downloads` directory, such as `halfnormal_plot.pdf` (transition height) and `genextreme_plot.pdf` (mission range), which can be included directly in reports or presentations.

### Example plotting workflow

After running the main optimisation / MCS pipeline and obtaining baseline and RBDO/MCS result folders, a typical post‑processing workflow is:

1. Ensure `input/environment_variables_plotting.yml` points `baseline.dir` and `mcs.dir` to the appropriate result directories.
2. Generate baseline MCS metrics:

   ```bash
   python plot_scripts/create_MCS_metric.py
   ```

3. Produce convergence plots, sorted mission‑sample tables, and correlation matrices:

   ```bash
   python plot_scripts/plot_mcs.py
   ```

4. Visualise RBDO optimisation and baseline performance:

   ```bash
   python plot_scripts/plot_parameters.py
   ```

5. Compare stochastic (MCS) and deterministic results:

   ```bash
   python plot_scripts/plot_comparison.py
   ```

6. (Optional) Inspect the assumed stochastic mission‑parameter distributions:

   ```bash
   python plot_scripts/plot_stochastic_mission_parameters.py
   ```
